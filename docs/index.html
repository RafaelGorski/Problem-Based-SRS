üéØ Overview

Problem-Based SRS is a continuation of research by Gorski and Stadzisz on improving software requirements specification quality. The methodology addresses a critical challenge: accurately capturing business intents of stakeholders, which remains a major factor in software project failures.

This repository provides standalone prompts and guidance for following a Problem-Based Software Requirements Specification methodology, designed for integration with AI agents like GitHub Copilot and Claude Code.

üìä The 5-Step Methodology

The Problem-Based SRS methodology consists of five structured steps that systematically analyze business context and specify software requirements:

Step 1: Customer Problems (CP)

Identify and classify customer problems by severity: Obligations (must solve), Expectations (should solve), Hopes (nice to solve)

prompts/CP.md

Step 2: Software Glance

Create a high-level view of the solution approach without detailed design

prompts/glance.md

Step 3: Customer Needs (CN)

Specify what the system must do to solve the identified problems

prompts/CN.md

Step 4: Software Vision

Define features, architecture, and technical constraints

prompts/vision.md

Step 5: Functional Requirements (FR)

Generate detailed, traceable functional requirements

prompts/FR.md

üöÄ Quick Start for Engineers

For New Features

When starting a new feature, use the AI agent with each prompt:

1. Identify Problems
@workspace Using prompts/CP.md, identify customer problems for [feature name]
Business Context: [describe current situation]

2. Design Solution Glance
@workspace Using prompts/glance.md, create a software glance
Input: [your CPs from step 1]

3. Specify Needs
@workspace Using prompts/CN.md, specify customer needs
Input: [CPs and Software Glance]

4. Define Vision
@workspace Using prompts/vision.md, create software vision
Input: [CPs, Glance, CNs]

5. Generate Requirements
@workspace Using prompts/FR.md, generate functional requirements
Input: [all previous artifacts]

Starting from Scratch

If you're new to a project or have only a business idea:

@workspace Use prompts/problem-based-SRS.md to guide me through requirements

Validating Existing Requirements

To check consistency between artifacts:

@workspace Using prompts/zigzag.md, validate traceability between my CPs, CNs, and Requirements
